# Prompt Templates Library

## Introduction

This library provides ready-to-use prompt templates for common tasks. Each template is designed following best practices from the prompting basics series. Simply fill in the bracketed placeholders with your specific information.

## How to Use These Templates

1. **Choose** a template matching your task
2. **Customize** by filling in the [placeholders]
3. **Adjust** optional sections as needed
4. **Test** and refine for your specific use case
5. **Save** successful variations for reuse

## Learning & Explanation Templates

### Template 1: Explain a Concept

```
I'm a [your background/level] learning about [topic].

I understand [what you already know] but I'm confused about [specific area].

Please explain [specific concept]:
- Use [analogies/examples/diagrams/etc.]
- Keep it at [beginner/intermediate/advanced] level
- Focus on [specific aspect if any]
- Format as [paragraphs/bullet points/step-by-step]

[Optional: Include a practice question to test understanding]
```

**Example:**
```
I'm a business student with no programming background learning about databases.

I understand that databases store information in tables, but I'm confused 
about how tables relate to each other.

Please explain database relationships:
- Use real-world business examples (like customers and orders)
- Keep it at beginner level
- Focus on one-to-many and many-to-many relationships
- Format as step-by-step explanation with examples

Include a practice question to test my understanding.
```

### Template 2: Step-by-Step Tutorial

```
Create a step-by-step tutorial for [task].

Target audience: [description of learner]
Prerequisites: [what they should know]
Goal: [what they'll be able to do]

Each step should include:
- What to do (clear action)
- Why it matters
- Expected result
- Common mistakes to avoid

Format: [numbered steps/phases/progressive complexity]
Length: [approximate number of steps or time]
```

### Template 3: Compare and Contrast

```
Compare [Option A] vs [Option B] for [use case/context].

I need to decide between them for [specific purpose].

Please compare:
- [Aspect 1]
- [Aspect 2]
- [Aspect 3]

For each aspect, explain:
- How they differ
- Pros and cons
- Which is better in what situation

End with a recommendation for my specific case: [your context details]
```

## Problem-Solving Templates

### Template 4: Debugging Assistance

```
I'm debugging [language/technology] code that [description of what it should do].

Problem: [describe the issue]
Expected: [what should happen]
Actual: [what actually happens]
Error message (if any): [paste error]

[paste code here]

Context:
- [Relevant details: environment, versions, dependencies]
- What I've already tried: [list attempts]

Please:
1. Identify the root cause
2. Explain why it's happening
3. Provide the fix with explanation
4. Suggest how to prevent similar issues

```

## Advanced Templates

### Template 16: Multi-Step Project

```
Help me plan and execute: [project description]

Current state: [where you are]
Desired state: [where you want to be]
Timeline: [when needed]
Resources: [what you have]

Please provide:

Phase 1 - Planning:
- Break down into main milestones
- Identify dependencies
- Estimate effort for each part

Phase 2 - Execution Strategy:
- Prioritized action steps
- Quick wins vs. long-term work
- Risk mitigation

Phase 3 - Success Metrics:
- How to measure progress
- Definition of done
- Quality checkpoints

Start with Phase 1, then we'll proceed step by step.
```

### Template 17: Chain of Thought Analysis

```
Analyze [problem/situation] using systematic reasoning.

Background: [provide context]
Question: [what needs to be determined]

Please think through this step by step:

1. Understanding:
   - Restate the problem in your own words
   - Identify what we know
   - Identify what we need to find

2. Analysis:
   - What approaches could work?
   - What are the key factors?
   - What assumptions are we making?

3. Solution:
   - Work through the logic
   - Show calculations/reasoning
   - Check for errors

4. Verification:
   - Does this answer make sense?
   - Alternative way to verify?
   - Confidence level and why?
```

### Template 18: Iterative Refinement

```
[Task description]

Process:

Draft 1: Provide initial attempt

Review 1: Critique your work
- What's strong?
- What's weak?
- What's missing?

Draft 2: Improved version based on review

Final Check:
- Does it meet all requirements?
- Is quality high enough?
- Any remaining issues?

Final Version: Polished output with summary of improvements
```

## Tips for Template Customization

1. **Start with structure**: Use template as-is first time
2. **Note what works**: Track successful modifications
3. **Create variations**: Save different versions for different contexts
4. **Combine templates**: Mix elements from multiple templates
5. **Build your library**: Collect templates that work for you
6. **Version your templates**: Improve them over time
7. **Share and learn**: Exchange templates with others
8. **Context matters**: Adjust formality and detail to your needs

## Template Variables Quick Guide

Common placeholders and what to fill in:

- `[topic]` - The subject matter
- `[task]` - What needs to be done
- `[context]` - Background information
- `[audience]` - Who this is for
- `[format]` - How to present it
- `[length]` - Size/duration constraints
- `[level]` - Complexity: beginner/intermediate/advanced
- `[tone]` - Style: formal/casual/technical/friendly
- `[requirements]` - Must-haves
- `[constraints]` - Limitations
- `[goal]` - Desired outcome
